# Avro fixtures ‚Äî what each file proves

All fixtures are synthetic, generated by `make_fixtures.py` in this directory
(deterministic: fixed values, fixed sync marker `powerquery-dless`). Regenerate with:

```
python3 -m venv venv && venv/bin/pip install fastavro
venv/bin/python make_fixtures.py
```

To test, load a fixture through `Avro.Document`:

```
Avro.Document(File.Contents("...\avro\test\scalars.avro"))
```

The decode algorithm has been cross-checked against fastavro (the Avro reference
implementation for Python) on every fixture: a mirror of the reader's exact logic
produces byte-identical results. The tables below are the expected Power Query output.

## scalars.avro ‚Äî 4 rows, codec null

Record `driverless.test.Scalars`. Proves: all primitive types, zigzag varint edge
values (`int` min/max, negative `long`, 2^52), float/double little-endian decode
(including negative zero and 1e300), empty string, empty bytes, UTF-8 text
(`h√©llo ‚úì`, embedded newline/tab), and the `["null","string"]` nullable idiom
(null, non-empty, and empty-string values).

| b | i | l | f | d | s | by | maybe |
|---|---|---|---|---|---|----|-------|
| true | 0 | 0 | 0 | 0 | (empty) | 0 bytes | null |
| false | -1 | 4503599627370496 | 1.5 | -2.75 | h√©llo ‚úì | 00 FF | present |
| true | 2147483647 | -4611686018427387904 | -0.5 | 1E+300 | line(lf)break | "avro" | null |
| false | -2147483648 | 1 | 100 | -0 | tab(tab)here | 01 | (empty) |

Column types: `b` logical, `i`/`l` Int64, `f`/`d` number, `s`/`maybe` text, `by` binary.

## scalars-deflate.avro ‚Äî 4 rows, codec deflate

Byte-for-byte the same rows as `scalars.avro`, compressed with the `deflate` codec
(raw RFC 1951, no zlib header ‚Äî must decompress with `Compression.Deflate`).
Proves the codec path; output must equal `scalars.avro` exactly.

## scalars-snappy.avro ‚Äî 4 rows, codec snappy

The same rows as `scalars.avro`, compressed with the `snappy` codec. Each Avro
snappy block is a raw snappy stream followed by a 4-byte big-endian CRC-32 of the
uncompressed block data. Proves the inlined codec oracle: the reader strips the
CRC trailer, wraps the snappy bytes in a one-cell Parquet file and lets
`Parquet.Document` decompress it. Output must equal `scalars.avro` exactly.

## scalars-zstandard.avro ‚Äî 4 rows, codec zstandard

The same rows as `scalars.avro`, compressed with the `zstandard` codec ‚Äî a plain
zstd frame per block, whose Frame_Content_Size the oracle reads to size the wrapper.
Proves the zstandard oracle path; output must equal `scalars.avro` exactly.

## empty.avro ‚Äî 0 rows

The `Scalars` schema with no data blocks at all (header + sync marker only).
Proves: an empty file yields an empty table that still has the schema's eight
typed columns, not an error and not a zero-column table.

## complex.avro ‚Äî 3 rows, codec null

Record `driverless.test.Complex`. Proves:

- `array<string>` and `array<long>`, including empty arrays (block count 0 first).
- `map<string,int>`, including an empty map. Maps decode to M records.
- `enum` (`Color`: RED/GREEN/BLUE) decodes to the symbol text.
- `fixed(4)` decodes to a 4-byte binary.
- Multi-branch union `["null","string","long"]` ‚Äî the three rows exercise all
  three branches (null, `"picked"`, `64`). Column stays untyped (`any`).
- Nested record `Point` decodes to an M record `[x, y]`.
- **Recursive named type**: `Node` references itself (`next: ["null","Node"]`).
  Row 2 carries a three-element chain 1 ‚Üí 2 ‚Üí 3; row 3 a single node (-7); row 1 null.
  This is the named-type-registry test ‚Äî a reader that compiles the schema eagerly
  without deferring named references will not terminate on this schema.

Column types: `dec` number, `day` date, `tmilli`/`tmicro` time,

## eventhubs.avro ‚Äî 4 rows, codec null

The schema Event Hubs Capture writes (`Microsoft.ServiceBus.Messaging.EventData`):
`SequenceNumber` long, `Offset` string, `EnqueuedTimeUtc` string,
`SystemProperties`/`Properties` maps of `["long","double","string","bytes"]`
unions, `Body` as `["null","bytes"]`. Proves the friendly path:

- Default: `Body` arrives as text ‚Äî two JSON payloads, one plain-text payload
  (`plain text body`), one null.
- `[ParseBodyAsJson = true]`: rows 1‚Äì2 become records
  (`deviceId`/`temperature`/`ok`), row 3 falls back to text (it is not JSON),
  row 4 stays null. With `[Strict = true]` as well, row 3 must error instead.
- `[KeepBodyBinary = true]`: `Body` stays binary.
- Map-of-union values decode (string and long branches in `SystemProperties`).

## multiblock.avro ‚Äî 200 rows, 9 data blocks, codec null

Record `driverless.test.Row` (`id` long, `name` string), written with a 256-byte
sync interval so the rows span 9 blocks. Proves: block iteration, per-block sync
marker validation, and row order across blocks (`id` must run 0..199 in order,
`name` = `row-0000`..`row-0199`). Also the fixture to use for `MaxRows` (e.g.
`[MaxRows = 5]` returns ids 0..4 without decoding later blocks).
