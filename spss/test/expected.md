# SPSS fixtures — what each file proves

All fixtures are synthetic, generated by `make_fixtures.py` in this directory.
Regenerate with:

```
python3 -m venv venv && venv/bin/pip install pyreadstat pandas
venv/bin/python make_fixtures.py
```

Most are written by pyreadstat (ReadStat — the reference implementation R haven
and Python use); two are hand-crafted byte by byte because ReadStat cannot
write them, then verified readable by pyreadstat. The parse logic destined for
`Spss.Document.pq` is mirrored in `mirror.py` and cross-validated cell-by-cell
against pyreadstat by `check_mirror.py` on every fixture (metadata included:
labels, formats, missing declarations, measures, value labels, encoding).

To test, load a fixture through `Spss.Document`:

```m
Spss.Document(File.Contents("...\spss\test\types.sav")){[Name = "Data"]}[Data]
```

## types.sav / rowcomp.sav / types.zsav — 4 cases, 5 variables

The same data in all three compression modes: `types.sav` uncompressed
(compression 0), `rowcomp.sav` bytecode-compressed (compression 1, what SPSS
writes by default), `types.zsav` zlib (compression 2, magic `$FL3`). All three
must produce identical output.

| id | score | big | str8 | str20 |
|---|---|---|---|---|
| 1 | -99 | 1E+300 | (empty) | (empty) |
| 2 | 151 | -2.75 | abc | twenty char string!! |
| 3 | 0.5 | 9007199254740992 | héllo ✓ | short |
| 4 | null | 0 | x | padded      end |

Proves: bytecode command bytes at the bias edges (-99 and 151 are codes 1 and
251), 253-literals (0.5, the `big` column), 255 system-missing (`score` row 4),
254 space runs (empty strings), string padding trim (trailing but not interior
spaces — `padded      end` keeps its interior run), UTF-8 text (`héllo ✓`),
and a string spanning multiple 8-byte elements (`str20`, width 20 = 3 elements).
Column types: `id`/`score`/`big` number, `str8`/`str20` text.

## labels.sav — 4 cases, 4 variables

The dictionary fixture. `Variables` must show:

| Name | Label | Measure | MissingValues |
|---|---|---|---|
| gender | Respondent gender | Nominal | 9 |
| rating | Satisfaction rating | Ordinal | -99 THRU -90 |
| vote | Voted yes/no | Nominal | |
| comment_text_very_long_name | Free-text comment | Nominal | |

`ValueLabels` must contain 7 rows: gender 1/2/9 → Male/Female/No answer,
rating 1/5 → Awful/Great, vote Y/N → Yes/No (string-valued labels).

Data: gender {1, 2, 1, 9}, rating {1, 5, -97, 3}, vote {Y, N, (empty), Y}.
Proves: variable labels, numeric and string value labels, discrete and range
user-missing declarations, measures, long variable names (subtype 13 — the
fourth column's name does not fit in 8 bytes), and the options:

- `[ApplyValueLabels = true]`: gender becomes {Male, Female, Male, No answer},
  rating {Awful, Great, -97, 3} (unlabeled values pass through).
- `[UserMissingToNull = true]`: gender row 4 (9) and nothing in rating (−97 is
  outside −99…−90) become null. Both options together: gender row 4 null.

## dates.sav — 4 cases, 3 variables

| d (DATE11 → date) | dt (DATETIME20 → datetime) | t (TIME8 → duration) |
|---|---|---|
| 2026-07-15 | 2026-07-15 13:45:30 | 13:45:30 |
| 1969-12-31 | 1900-01-01 00:00:01 | 00:00:00 |
| 1582-10-15 | 2000-02-29 23:59:59 | 23:59:59 |
| null | null | null |

Proves: the 1582-10-14 epoch (row 3's date is day one of the Gregorian
calendar), seconds-based datetimes, TIME as duration, and system-missing in
date columns.

## longstr.sav — 3 cases, 5 variables

Very long strings (extension record 14). Widths: 300 (2 segments), 255 (the
single-segment maximum), 256 (the smallest 2-segment case), 600 (3 segments),
plus a trailing numeric to prove element alignment after the segments.

Row 1: `A`×300, `C`×255, `E`×256, `G`×600, 1. Row 2: `b`×10, (empty), `f`×256,
(empty), 2. Row 3: (empty), `d`×254, (empty), `h`×300, 3.

Proves: segment stitching (255 data bytes per segment, leftover allocated
space ignored — the values must come back at exactly their written lengths
with no interior spaces), the 255/256 boundary, and that the numeric column
after the segments still decodes (element offsets survived the merge).
Variables must report widths 300/255/256/600 and formats A300/A255/A256/A600.

## empty.sav — 0 cases, 2 variables

Dictionary but no data. Must yield an empty `Data` table that still has both
typed columns (`a` number, `b` text), not an error and not a zero-column table.

## legacy1252.sav — 4 cases, 2 variables (hand-crafted)

The shape of a pre-Unicode SPSS file: no encoding record (subtype 20), only
charcode 1252 in the machine integer record; uncompressed data; codepage-1252
accented strings. `CITY` must decode to {Málaga, Zürich, León, (empty)} — if
these come back mojibake the charcode fallback is broken. `NUM` carries value
labels 1/2 → première/deuxième (cp1252 labels) and a system-missing in row 4.
Also proves value-label records (types 3/4) on a file with no subtype 13 —
column names are the raw short names `NUM`, `CITY`.

## multiblock.zsav — 120 cases, 2 variables (hand-crafted)

A zsav whose bytecode stream spans two zlib blocks (ReadStat always writes
one, so only a hand-built file exercises the block-descriptor walk). `N` must
run -99, -98, … 20 in order (all values inside the command-byte range); `S`
must be `blk-0000` … `blk-0119` in order. Proves: the 24-byte zlib header, the trailer descriptor
walk, per-block decompressed-size validation, and block-boundary reassembly
(the split lands mid-case).
