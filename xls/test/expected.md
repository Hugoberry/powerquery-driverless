# .xls fixtures — what each file proves

All fixtures are synthetic and licence-clean. `types.xls`, `multisheet.xls`,
`strings.xls` and `date1904.xls` are generated by `make_fixtures.py` with xlwt
(an independent BIFF8 writer); `edge.xls` is handcrafted byte-by-byte by
`make_edge_fixture.py` to cover records xlwt never emits. Regenerate with:

```
python3 -m venv venv && venv/bin/pip install xlwt xlrd
venv/bin/python make_fixtures.py .
venv/bin/python make_edge_fixture.py .
```

Every fixture has been cross-checked against xlrd (the Python reference reader
for BIFF): a Python mirror of the reader's exact parse logic produces output
identical to xlrd's, cell by cell, on all five files. The tables below are the
expected Power Query output.

To test, load a fixture through `Xls.Workbook`:

```
Xls.Workbook(File.Contents("...\xls\test\types.xls"))
```

## types.xls — sheet `Types`, 6 rows x 5 columns

Proves: LABELSST shared strings (ASCII, UTF-16 with CJK and accents, a 300-char
string), RK and NUMBER numerics, boolean cells, datetime / date / time-only /
pre-1900-03-01 date formats (builtin and custom), a formatted number that is
*not* a date (`0.00%`), an empty cell inside the used range, and formula cells
whose cached result is an empty string (xlwt writes result tag 3).

| Column1 | Column2 | Column3 | Column4 | Column5 |
|---|---|---|---|---|
| id | text | number | flag | when |
| 1 | plain ascii | 3.14159 | true | 2024-03-15 13:45:30 |
| 2 | unicodé – 你好 | -273.15 | false | 1999-12-31 00:00:00 |
| 3 | (null) | 1234567890 | true | 1899-12-31 06:30:00 |
| 4 | xxx…(300 chars) | 0.07 | false | 1900-02-28 00:00:00 |
| 5 | formulas | (empty text) | (empty text) | (empty text) |

Note the time-only cell (r3) surfaces as `1899-12-31 06:30:00` — serials below 1
carry no date part — and `1900-02-28` proves the pre-leap-bug epoch branch
(serials 1..59 use the 1899-12-31 epoch so dates display as Excel shows them).

## multisheet.xls — three sheets

`First` (2 rows), `Hidden Sheet` (hidden, 1 row), `Empty` (no cells). Proves:
multiple BOUNDSHEET records with correct substream offsets, hidden-sheet
filtering (`Hidden Sheet` appears only with `[IncludeHiddenSheets = true]`, and
its `Hidden` column is true), and that an empty sheet yields an empty table,
not an error.

## strings.xls — sheet `Strings`, 401 rows x 2 columns

400 unique ~34-char strings push the SST past the 8224-byte record limit,
forcing two CONTINUE records, with strings split mid-text at both boundaries
(each continuation restarts with a fresh flags byte). Proves the SST
continuation logic — the single trickiest part of BIFF8. Row `i` is
`unique-string-{i:04d}-abcdefghijabcdefghijabcdefghij`; every value must come
back intact, no truncation, no shifted indexes.

## date1904.xls — sheet `Dates1904`, 3 rows x 1 column

`DATEMODE 1`. Header row, then `2024-03-15 13:45:30` and `1904-01-02 00:00:00`
(serial 1). Proves the 1904 date system epoch.

## edge.xls — sheet `Edge`, 6 rows x 5 columns

Handcrafted to cover what xlwt cannot write. Also the only fixture whose
Workbook stream is under 4096 bytes, so it lives in the CFB **mini-stream**
(miniFAT chain inside the root entry's storage) — the container path Desktop
tests rarely exercise.

| Column1 | Column2 | Column3 | Column4 | Column5 |
|---|---|---|---|---|
| 30000 | -5 | 123.45 | 3.5 | 3.14158203125 |
| (null) | (null) | (null) | (null) | 2023-03-15 00:00:00 |
| inline latin | inline ünïcode € | from the shared string table | ünïcode ✓ | (null) |
| true | false | Error #DIV/0! | Error #VALUE! | Error #N/A |
| 42.5 | cached formula string | true | Error #N/A | (null) |
| 2.718281828 | (null) | (null) | (null) | (null) |

Proves, row by row:

- r0 — a single MULRK record with all four RK encodings: integer, negative
  integer (30-bit two's complement), integer/100, truncated float, and
  truncated float/100. `3.14158203125` is correct: RK floats keep only the top
  30 bits of the IEEE double, and xlrd decodes the same value.
- r1 — BLANK and MULBLANK (null but extend the used range), and a date-styled
  RK integer (custom format `yyyy-mm-dd`, serial 45000).
- r2 — inline LABEL strings (compressed/Latin-1 and UTF-16 wide) next to
  LABELSST references (ASCII and non-Latin-1 text).
- r3 — BOOLERR both ways: booleans and real error cells. Error cells surface
  as cell-level errors in the table (matching `Excel.Workbook`), with the Excel
  error name in the message.
- r4 — FORMULA cached results of all four kinds: number, string (in the
  following STRING record), boolean, error.
- r5 — a plain 8-byte NUMBER record.
