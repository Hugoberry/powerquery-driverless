# MATLAB fixtures: what each file proves

All fixtures are synthetic, generated by `make_fixtures.py` in this directory.
Regenerate with:

```
python3 -m venv venv && venv/bin/pip install scipy numpy
venv/bin/python make_fixtures.py
```

Most are written by scipy.io.savemat, the de facto open reference writer.
Four are hand-crafted byte by byte because scipy will not emit them (numeric
storage compression, miUINT16 char data, a big-endian file, a truly empty
file). The parse logic destined for `Matlab.Document.pq` is mirrored in
`mirror.py` and cross-validated cell-by-cell against `scipy.io.loadmat` (the
reader of record) by `check_mirror.py` on every fixture.

To test, load a fixture through `Matlab.Document`:

```m
Matlab.Document(File.Contents("...\matlab\test\matrix.mat")){[Name = "m_double"]}[Data]
```

## scalars.mat: 8 variables, uncompressed

One variable per scalar and vector kind. `d_scalar` 3.14159, `i_scalar` -42
(int32, written as a small data element), `u8_scalar` 200, `single_scalar`
1.5, `bool_scalar` true (uint8 class with the logical flag: the nav table
must show class `logical` and the value must be the logical `true`, not 1),
`str_var` the text "hello world", `row_vec` the list {1, 2, 3, 4}, `col_vec`
the list {10, 20, 30}.

Proves: 1x1 arrays collapse to scalars, row and column vectors both become
lists, char row vectors become text, the logical flag wins over the numeric
class, and the small data element format parses.

## types.mat: 10 variables, uncompressed

Every numeric class as a 1x3 vector at its extremes: int8 {-128, 0, 127},
uint8 {0, 128, 255}, int16 {-32768, 0, 32767}, uint16 {0, 40000, 65535},
int32 {-2147483648, 0, 2147483647}, uint32 {0, 3000000000, 4294967295},
int64 and uint64 up to 9007199254740000 either side of zero (inside the
2^53 exactness limit), single {-1.5, 0, 2.25}, double {-1.5, 0, 2.25}.

Proves: all ten numeric storage widths decode with correct signedness in
both halves of each range.

## matrix.mat: 3 variables, uncompressed

`m_double` 2x3 [[1 2 3],[4 5 6]] must become a 2-row table with columns
C1..C3 (the file stores it column-major; the table must read row-major).
`m_int` 3x2 int32 [[1 2],[3 4],[5 6]], typed Int64. `m_3d`, a 2x2x2 double
holding 0..7, must become a long table D1, D2, D3, Value with D1 varying
fastest: rows (1,1,1,0), (2,1,1,4), (1,2,1,2), (2,2,1,6), (1,1,2,1), ...

Proves: column-major to row-major transposition, integer vs floating column
typing, and the N-dimensional long-table shape.

## compressed.mat: 3 variables, zlib-compressed (the v7 default)

`m_double` and `m_int` identical to matrix.mat, plus `greeting`
"compressed hello". Every top-level element is miCOMPRESSED: a zlib stream
(2-byte header, deflate body, Adler-32 trailer) that the reader must strip,
inflate with `Compression.Deflate`, and parse as a normal matrix element.

Proves: the v7 compressed container produces exactly the same output as the
uncompressed layout.

## struct.mat: 2 variables

`person`, a 1x1 struct, must become the record
[name = "Ada", age = 36, scores = {9, 8.5, 7}]. `arr`, a 1x2 struct array
with fields x and y, must become a 2-row table: (1, "first"), (2, "second").

Proves: scalar structs collapse to records, struct arrays become one row
per element with one column per field, and the field-name subelement
(name length, packed name list) parses.

## cell.mat: 2 variables

`mixed`, a 1x3 cell {"alpha", 42, [1 2 3]}, must become the list
{"alpha", 42, {1, 2, 3}}: heterogeneous, elements converted recursively.
`words`, a 3x1 cell, must become the list {"one", "two", "three"}.

Proves: cell arrays shape like numeric arrays and recurse into nested
matrix elements.

## complex.mat: 2 variables

`c_scalar` 3+4i must become the record [Re = 3, Im = 4]. `c_mat` 2x2 must
become a table whose cells are [Re, Im] records: row 1 (1+1i, 2-2i),
row 2 (3+0i, 0+5i).

Proves: the complex flag triggers the second (imaginary) data element and
both parts pair up cell by cell.

## sparse.mat: 1 variable

`sp`, a 3x3 CSC sparse double with nonzeros (2,1)=4, (1,3)=3, (3,3)=6, must
become the table {Row, Column, Value} with exactly those three rows in
column-major order: (2,1,4), (1,3,3), (3,3,6).

Proves: ir/jc/pr decoding, 1-based indices, and that only stored nonzeros
appear.

## unicode.mat: 2 variables

`greeting` "café naïve", stored as miUTF8 by scipy. `charmat`, a 2x3 char
matrix stored column-major as "adbecf", must come back as the list
{"abc", "def"}.

Proves: UTF-8 char data and char-matrix de-interleaving.

## widechar.mat: 1 variable, hand-crafted

`wide` "Ωmega™" stored as miUINT16 (UTF-16 code units), the form real
MATLAB writes and scipy never emits. scipy also misreads this file with its
default codec, so `check_mirror.py` passes `uint16_codec` explicitly.

Proves: the miUINT16 char path.

## bigend.mat: 3 variables, hand-crafted big-endian

Header endian indicator "MI". `bd` 2x2 double [[1.5 -2.5],[3.0 4.25]],
`bi` 1x3 int16 {-300, 0, 300}, `bc` "abΩ" as big-endian miUINT16.

Proves: every multi-byte read honors the byte order: tags, small-element
tags (the two 16-bit halves swap), dimensions, doubles, int16, and UTF-16
char data.

## packed.mat: 1 variable, hand-crafted

`p`, a 1x3 array of class double whose data element is stored as miINT8
{10, 20, 30}. This is MATLAB's numeric storage compression, which scipy
never writes. It must come back as the numeric list {10, 20, 30}: the class
decides the type, the storage decides the bytes. Also exercises a small
data element for the name.

Proves: storage type and array class are independent.

## empty.mat: 0 variables, hand-crafted

A valid 128-byte header and nothing else. Must produce an empty navigation
table, not an error.
