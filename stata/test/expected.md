# Stata fixtures — what each file proves

All fixtures are synthetic, generated by `make_fixtures.py` in this directory.
Regenerate with:

```
python3 -m venv venv && venv/bin/pip install pyreadstat pandas
venv/bin/python make_fixtures.py
```

Most are written by pandas (whose Stata writer preserves all five numeric
storage types and writes strLs) or pyreadstat (ReadStat — the reference
implementation R haven and Python use; it writes every dta version and
extended missing values); two are hand-crafted byte by byte because neither
library can write them, then verified readable by pandas. The parse logic
destined for `Stata.Document.pq` is mirrored in `mirror.py` and
cross-validated cell-by-cell against pandas and pyreadstat by
`check_mirror.py` on every fixture (metadata included: variable labels,
formats, value labels, missing codes, dataset label, sort order, date
conversion).

To test, load a fixture through `Stata.Document`:

```m
Stata.Document(File.Contents("...\stata\test\types118.dta")){[Name = "Data"]}[Data]
```

## types114 / types117 / types118 / types119 — 3 obs, 6 variables

The same data in four container versions. 114 is the legacy raw layout,
117 the first tag-delimited one, 118 the current default, 119 the wide-K
variant (4-byte variable count and sortlist entries). All four must produce
identical output.

| vbyte | vint | vlong | vfloat | vdouble | vstr |
|---|---|---|---|---|---|
| -127 | -32767 | -2147483647 | -1.5E38 | 8.988E307 | abcdefgh |
| 100 | 32740 | 2147483620 | 1.5E38 | -8.988E307 | sp␠ |
| 0 | 1 | 2 | 0.25 | null | (empty) |

Proves: all five storage types at their documented valid-range edges (one
step past any of these integer values is a missing code), system missing in
a double, strf handling — `abcdefgh` is exactly the declared width str8 (no
terminator in the file), `sp ` keeps its significant trailing space, and the
empty string decodes empty. Column types: integers `Int64.Type`, floats
`number`, `vstr` `text`.

## types113 / types115 — 3 obs, 3 variables

The two legacy containers pandas cannot write (written by ReadStat, which
promotes small types, so these only prove the container differences):
ds_format bytes 0x71/0x73 and the 12-byte (113) versus 49-byte (115)
display-format field. Data: `num` {1.5, -2.75, 1E300}, `count` {1, 2, 3},
`name` {alpha, (empty), z}.

## labels118 — 4 obs, 3 variables

The dictionary fixture. `Variables` must show labels "Respondent gender",
"Satisfaction rating", "Free-text note"; `gender` and `rating` reference
value-label sets. `ValueLabels` must contain gender 1/2 → Male/Female and
rating 1/5 → Awful/Great. The dataset label is "driverless label fixture".

Data: gender {1, 2, 1, .a}, rating {1, 5, .z, .}, note {fine, bad, (empty),
ok}. Proves the missing scheme end to end: by default gender row 4 and
rating rows 3-4 are null; with `[ExtendedMissingToText = true]` gender row 4
is ".a" and rating row 3 is ".z" (rating row 4 stays null — it is `.`); with
`[ApplyValueLabels = true]` gender becomes {Male, Female, Male, null} and
rating {Awful, Great, null, null} (the missings carry no labels here).

## dates118 — 4 obs, 8 variables

Every %t flavor, stored as plain numbers with only the display format saying
they are dates. Row 4 is all system-missing (null in every column).

| Variable | Format | Row 1 | Row 2 | Row 3 |
|---|---|---|---|---|
| td | %td | 2026-07-15 | 1960-01-01 | 1959-12-31 |
| tc | %tc | 2026-07-15 13:45:30.123 | 1960-01-01 00:00:00 | 1960-01-01 23:59:59.999 |
| tm | %tm | 2026-07-01 | 1960-01-01 | 1959-12-01 |
| tq | %tq | 2026-07-01 | 1960-01-01 | 1959-10-01 |
| th | %th | 2026-07-01 | 1960-01-01 | 1959-07-01 |
| tw | %tw | 2026-07-16 | 1960-01-01 | 1960-12-23 |
| ty | %ty | 2026 | 1960 | 1900 |
| dold | %d | 1960-01-01 | 1960-04-10 | 1959-09-23 |

Proves: the 1960-01-01 epoch in days, milliseconds, months, quarters,
half-years and weeks; negative values (dates before 1960); the old-style %d
notation; %ty staying numeric; missing-before-conversion ordering (row 4
must be null, not a date near year 22808).

## strl117 / strl118 — 4 obs, 3 variables

strLs behind (v,o) pointers: a 3000-byte string (longer than any strf can
hold), an empty string (recorded as pointer (0,0) with no GSO), a short
string, and a repeat of the long one. `n` trails the strL column to prove
row alignment past the 8-byte pointer. `su` is a plain strf ("héllo ✓" in
118; latin-1 "héllo" in 117) to prove encoding.

strl117 proves the 117 pointer split (4+4 bytes, 4-byte GSO offsets);
strl118 proves the 118 one (2+6 bytes, 8-byte GSO offsets).

## empty118 — 0 obs, 2 variables

Dictionary but no data. Must yield an empty `Data` table that still has both
typed columns (`a` number, `b` text), not an error and not a zero-column
table.

## legacy1252.dta — 4 obs, 3 variables (hand-crafted)

The shape of a pre-Unicode Stata file (format 114, Windows): codepage-1252
accents in the dataset label ("légacy fixture"), a variable label
("Numéro"), string data (Málaga, Zürich, León) and value labels
(prémier/deuxième) — if any of these come back mojibake the legacy-encoding
default is broken. Also proves: two expansion fields that must be skipped by
their 5-byte headers, a sort order (`Stata.SortedBy` = {grp}), `.` stored in
a double, and `.a` stored in a byte (grp row 4, value 102).

## msf118.dta — 4 obs, 3 variables (hand-crafted)

A big-endian (MSF) format-118 file — old Solaris/AIX Stata wrote these, no
current library writes them, and ReadStat refuses to read them (pandas
reads it, and validated this fixture). Every integer, double, strL pointer
and label table in the file is byte-swapped. `id` is sorted, labeled by set
`idlbl`, and carries `.a` in row 3 (which has the value label "refused" —
a label on an extended missing code, ValueLabels row {id, .a, refused}) and
`.` in row 4. The strL column has a GSO-backed value, a cross-link to it
from row 2, an empty (0,0), and a binary (t=129) GSO (`P\0NG`) that must
come back as a binary value, leaving the column untyped. The single
characteristic's contents contain the literal string "</characteristics>"
to punish readers that scan for the close tag instead of walking <ch>
lengths.

With `[ApplyValueLabels = true]`, `id` must read {one, two, refused, null}.
