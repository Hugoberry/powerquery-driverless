# .xlsb fixtures — what each file proves

All fixtures are synthetic and licence-clean, generated by `make_fixtures.py`
in this directory with a purpose-built BIFF12 writer. Regenerate with:

```
python3 -m venv venv && venv/bin/pip install pyxlsb
venv/bin/python make_fixtures.py .
```

Every fixture has been cross-checked against pyxlsb (an independent .xlsb
reader): a Python mirror of the reader's exact parse logic agrees with pyxlsb
cell by cell. The one exception is inline-string cells (`BrtCellSt`, record
0x0006), which pyxlsb skips entirely — those cells carry known constant values
from the writer instead. The tables below are the expected Power Query output.

To test, load a fixture through `Xlsb.Workbook`:

```
Xlsb.Workbook(File.Contents("...\xlsb\test\types.xlsb"))
```

## types.xlsb — sheet `Types`, 6 rows x 5 columns

| Column1 | Column2 | Column3 | Column4 | Column5 |
|---|---|---|---|---|
| shared one | shared twö ✓ | inline string € | (null) | (null) |
| 30000 | -5 | 123.45 | 3.5 | 3.14158203125 |
| 2.718281828 | 0.07 | 2023-03-15 00:00:00 | 2023-03-15 13:45:30 | (null) |
| (all nulls — row 3 is skipped in the file) | | | | |
| true | false | Error #DIV/0! | Error #N/A | (null) |
| 42.5 | cached formula string | true | Error #N/A | (null) |

Proves, row by row:

- r0 — shared strings via `BrtCellIsst` (ASCII and non-ASCII), an inline
  `BrtCellSt` string, and a styled `BrtCellBlank` (null, but extends the grid
  to 4+ columns).
- r1 — `BrtCellRk` with all four RK encodings: integer, negative integer
  (30-bit two's complement), integer/100, truncated float, truncated
  float/100. `3.14158203125` is correct — RK floats keep only the top 30 bits
  of the double.
- r2 — `BrtCellReal` full doubles; a percent-formatted number that must *not*
  become a date; a date-styled RK (builtin format 14) and a datetime-styled
  real (custom format `yyyy-mm-dd hh:mm:ss`, serial 45000.573264).
- r3 — absent from the file (no BrtRowHdr): must appear as an all-null row,
  not be silently dropped.
- r4 — `BrtCellBool` both ways and `BrtCellError` cells. Error cells surface
  as cell-level errors in the table (matching `Excel.Workbook`), with the
  Excel error name in the message.
- r5 — cached formula results of all four kinds: `BrtFmlaNum`,
  `BrtFmlaString`, `BrtFmlaBool`, `BrtFmlaError`.

## multisheet.xlsb — three sheets

`First` (2 rows: `multi`, 1), `Hidden Sheet` (hidden, one inline string),
`Empty` (no cells). Proves: multiple `BrtBundleSh` entries resolved through
`xl/_rels/workbook.bin.rels`, hidden-sheet filtering (`Hidden Sheet` appears
only with `[IncludeHiddenSheets = true]`, its `Hidden` column true), and that
an empty sheet yields an empty table, not an error.

## date1904.xlsb — sheet `Dates1904`, 1 row x 2 columns

`BrtWbProp` has the 1904 flag set. Expected: `2024-03-15 13:45:30` (serial
43903.573264) and `1904-01-02 00:00:00` (serial 1). Proves the 1904 epoch is
read from the workbook part and applied to both real and RK serials.

## stored.xlsb — sheet `Stored`, 1 row x 2 columns

Identical structure to the others but every ZIP entry uses method 0 (stored)
instead of deflate. Expected: `stored, not deflated` and `99.5`. Proves the
stored-entry path through the central directory (no `Binary.Decompress`).
